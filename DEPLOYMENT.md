OpenAI API Key\n- Go to https://platform.openai.com/api-keys\n- Create a new API key\n- Add to `backend/.env`: `OPENAI_API_KEY=sk-...`\n\n### 2. Twitter API (Bearer Token)\n- Go to https://developer.twitter.com/\n- Create a new app\n- Generate Bearer Token\n- Add to `backend/.env`: `TWITTER_BEARER_TOKEN=...`\n\n### 3. Reddit API\n- Go to https://www.reddit.com/prefs/apps\n- Create a new app (script type)\n- Add credentials to `backend/.env`:\n  - `REDDIT_CLIENT_ID=...`\n  - `REDDIT_CLIENT_SECRET=...`\n\n### 4. Google News API\n- Go to https://newsapi.org/\n- Get free API key\n- Add to `backend/.env`: `GOOGLE_NEWS_API_KEY=...`\n\n## MongoDB Setup\n\n### Local MongoDB\n```bash\n# macOS with Homebrew\nbrew install mongodb-community\nbrew services start mongodb/brew/mongodb-community\n\n# Ubuntu/Debian\nsudo apt install mongodb\nsudo systemctl start mongodb\n```\n\n### MongoDB Atlas (Cloud)\n1. Go to https://www.mongodb.com/atlas\n2. Create free cluster\n3. Get connection string\n4. Update `backend/.env`: `MONGODB_URI=mongodb+srv://...`\n\n## Environment Variables\n\n### Backend (.env)\n```bash\nPORT=5000\nNODE_ENV=development\nFRONTEND_URL=http://localhost:3000\n\n# Database\nMONGODB_URI=mongodb://localhost:27017/brandtracker\n\n# API Keys (Required)\nOPENAI_API_KEY=your_openai_api_key_here\nTWITTER_BEARER_TOKEN=your_twitter_bearer_token_here\nREDDIT_CLIENT_ID=your_reddit_client_id_here\nREDDIT_CLIENT_SECRET=your_reddit_client_secret_here\nGOOGLE_NEWS_API_KEY=your_google_news_api_key_here\n\n# Email Configuration (Optional)\nSMTP_HOST=smtp.gmail.com\nSMTP_PORT=587\nSMTP_USER=your_email@gmail.com\nSMTP_PASS=your_app_password_here\n\n# JWT Secret\nJWT_SECRET=your_super_secret_jwt_key_here\n```\n\n### Frontend (.env)\n```bash\nREACT_APP_API_URL=http://localhost:5000/api\nREACT_APP_SOCKET_URL=http://localhost:5000\nREACT_APP_ENV=development\n```\n\n## Development\n\n### Start Development Server\n```bash\n# Start both frontend and backend\nnpm run dev\n\n# Or start individually\nnpm run server  # Backend only\nnpm run client  # Frontend only\n```\n\n### Available Scripts\n```bash\n# Root package.json\nnpm run dev         # Start both frontend & backend\nnpm run server      # Start backend only\nnpm run client      # Start frontend only\nnpm run build       # Build frontend for production\nnpm run install-deps # Install all dependencies\n\n# Backend\ncd backend\nnpm start           # Start server\nnpm run dev         # Start with nodemon\nnpm test            # Run tests\n\n# Frontend\ncd frontend\nnpm start           # Start React dev server\nnpm run build       # Build for production\nnpm test            # Run tests\n```\n\n## Production Deployment\n\n### Docker Deployment\n```bash\n# Build backend image\ncd backend\ndocker build -t brand-tracker-api .\n\n# Build frontend image\ncd ../frontend\ndocker build -t brand-tracker-web .\n\n# Run with Docker Compose\ndocker-compose up -d\n```\n\n### Manual Deployment\n1. Set up production MongoDB\n2. Configure production environment variables\n3. Build frontend: `npm run build`\n4. Start backend: `npm start`\n5. Serve frontend build with nginx/apache\n\n### Environment Setup\n```bash\n# Production environment variables\nexport NODE_ENV=production\nexport PORT=5000\nexport MONGODB_URI=\"mongodb://...\"\nexport OPENAI_API_KEY=\"...\"\n# ... other API keys\n```\n\n## Monitoring & Health Checks\n\n### Health Check Endpoints\n- Backend Health: `GET /api/health`\n- Database Status: Check MongoDB connection logs\n- WebSocket Status: Check browser console for connection\n\n### Logging\n- Backend logs to console (use PM2 or similar for production)\n- Frontend errors visible in browser console\n- API errors returned in standardized format\n\n## Troubleshooting\n\n### Common Issues\n\n1. **MongoDB Connection Failed**\n   - Check if MongoDB is running\n   - Verify connection string in `.env`\n   - Check network connectivity for Atlas\n\n2. **API Rate Limits**\n   - Twitter: 300 requests per 15 minutes\n   - Reddit: 60 requests per minute\n   - OpenAI: Depends on your plan\n   - Google News: 1000 requests per day (free)\n\n3. **CORS Issues**\n   - Check FRONTEND_URL in backend `.env`\n   - Ensure ports match between frontend and backend\n\n4. **WebSocket Connection Issues**\n   - Check if backend is running\n   - Verify SOCKET_URL in frontend `.env`\n   - Check firewall settings\n\n### Debug Mode\n```bash\n# Enable debug logging\nexport DEBUG=brand-tracker:*\nnpm run dev\n```\n\n## API Documentation\n\n### Endpoints\n- `GET /api/health` - Health check\n- `GET /api/mentions` - Get mentions with filters\n- `GET /api/analytics/dashboard` - Dashboard statistics\n- `GET /api/alerts` - Get alert configurations\n- `POST /api/alerts` - Create new alert\n\n### WebSocket Events\n- `newMention` - Real-time mention updates\n- `alertTriggered` - Alert notifications\n\n## Security Considerations\n\n- All API keys should be kept secret\n- Use HTTPS in production\n- Implement rate limiting\n- Validate all user inputs\n- Use environment variables for sensitive data\n\n## Performance Optimization\n\n- MongoDB indexing for faster queries\n- Redis for caching (optional)\n- CDN for frontend assets\n- Load balancing for multiple instances\n\n---\n\n**Need Help?** Check the troubleshooting section or create an issue in the repository.
